# Contributing to Kala

Thank you for your interest in contributing to Kala! We welcome all contributions including bug reports, feature requests, documentation improvements, and code changes.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/kala.git`
3. Install dependencies: `npm install`
4. Create a new branch for your feature or bugfix: `git checkout -b feature/your-feature-name`

## Development Workflow

- We use Expo for development. Run `npm start` to open the Expo development server.
- Ensure your code follows the existing style and conventions.
- If you add a new feature, please update the documentation where necessary.

## Submitting a Pull Request

1. Push your branch to your fork: `git push origin feature/your-feature-name`
2. Open a Pull Request against the `main` branch.
3. Provide a clear description of your changes and why they are needed.

We will review your PR as soon as possible. Thank you for making Kala better!
